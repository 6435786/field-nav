## מה הושלם בשיחה הזו (סדר זמני)

### Phase 1 — תשתית הרשאות
- `nav_user_roles/{uid}` נוצר אוטומטית בכניסה ראשונה עם role:'user'
- Helpers: `isApproved()` / `isAdmin()` / `isSuperAdmin()` (גם בקליינט וגם ב-Firestore rules)
- חוקי Firestore בסיסיים
- gating: רמה 1 → "התחבר עם גוגל" להקלטה/שמירה; רמה 2 → לא יכולה ליצור פרוייקטים

### Phase 2 — ניהול אדמין
- כפתור ⚙️ ניהול ב-header (לרמה 4+)
- מודל ניהול עם 2 לשוניות: משתמשים / קבוצות
- עריכת משתמש: role + שם + טלפון + allowPrivatePoints + קבוצות
- יצירה/עריכה/מחיקה של קבוצות (סופר-אדמין בלבד)

### Phase 3 — הרשאות פרוייקטים
- מודל חדש: `viewerGroups`/`editorGroups`/`viewerUsers`/`editorUsers`/`openToAdd`/`deniedUsers`
- Multi-picker UI עם chips + search-autocomplete + הוספה ידנית למייל
- **Editor כופה Viewer** (chip עם 🔒 בצפייה אם בעריכה)
- ניהול קבוצה מורחב: 3 לשוניות (כללי / חברים / פרוייקטים), עריכת חברים + פרוייקטים עם דרגות, ניקוי בכריזון (`arrayRemove` ב-batch)
- broadcast doc `nav_meta/projects_signal` — מתעדכן בכל שינוי פרוייקט, גורם לקליינטים לרענן

### Phase 4 חלקית
- **#1 פרופיל אישי:** לחיצה על השם/אווטר ב-header → מודל "👤 הפרופיל שלי" (שם + טלפון). זמין לכל מחובר.
- **#2 שורת "הוסיף:" בפאנל הפרטים:** SVG אמיתי של 📞 (ירוק קלאסי) + לוגו WhatsApp (ירוק WhatsApp). הופיע על נקודות שמורות עם createdBy. דורש את הקלת קריאה ל-`nav_user_roles` ל-`signedIn()`.
- **#4 הערות לנקודה:** שדה `notes` בכל נקודה, textarea במודלי הוספה/עריכה, קופסה צהובה ("sticky note") בפאנל הפרטים.

### באגים שתוקנו בדרך
- **כניסה אנדרואיד נשברה:** signInWithRedirect לא חזר → עברנו ל-`signInWithPopup` עם fallback ל-redirect
- **דאמני "מפות אופליין"** הוסתר גם מסופר-אדמין → תוקן (תמיד יוצרים את ה-overlay בבקרה, מסתירים ב-CSS לפי `body.offline-overlay-hidden`)

- **role עצמי אבד:** המשתמש מחק בטעות את המסמך שלו (בבדיקות "משתמש חדש"), הקוד נתן user; הוא תיקן ידנית ב-Firestore Console

### פיצ'רים נוספים בשיחה

## **מה מחכה ברגע זה (סדר עדיפויות)**

### B — סיבוב מפה + 3 מצבים (הוסכם עם המשתמש, ממתין לביצוע)

**מצבים:**
- **צפון** (bearing=0, ברירת מחדל)
- **ידני** (bearing מסיבוב באצבעות)
- **מצפן** (bearing לפי המצפן של המכשיר)

**מעברים (חשוב לדייק — המשתמש תיקן אותי כמה פעמים על זה):**
- **🧭 כפתור מצפן בעמודה השמאלית:**
  - צפון → מצפן
  - ידני → צפון
  - מצפן → צפון
  - (כלומר: מצפון הולכים למצפן; מכל מצב אחר חוזרים לצפון)
- **סיבוב באצבעות:** כל מצב → ידני
- **🎯 מיקום נוכחי:** תמיד מפעיל **מעקב** (auto-center). הסיבוב נקבע לפי הכניסה:
  - אם נכנס ממצב צפון → נשאר צפון + מעקב
  - אם נכנס מידני או ממצפן → עובר למצפן + מעקב

**כיבוי מעקב:** **גם** לחיצה נוספת על 🎯 **וגם** גרירת המפה ידנית (אופציה ג של המשתמש).

**מימוש טכני:**
- הוספת ספריית `leaflet-rotate` (כ-10KB) — Leaflet לא תומך בסיבוב מובנית
- הוספת כפתור מצפן 🧭 בעמודה השמאלית (ליד 🎯 והקלטה). חץ קטן שמסתובב עם ה-bearing.
- שינוי בכפתור 🎯: עכשיו "show once" — צריך להפוך ל"follow + maybe compass"
